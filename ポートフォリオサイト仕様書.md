```css
  --letterbox:#000000; /* レターボックスの黒帯 */
  --ink:#ECE7DF;       /* 文字（温白） */
  --ink-mut:#8C887F;   /* 補足グレー */
  --line:#262A30;      /* ヘアライン */
  --amber:#E9A94C;     /* 唯一のアクセント（プロジェクター光） */
  --amber-dim:#6b5430;
  --r:2px;             /* 角丸 */

```

### フォントの使い分け（厳守）

読み込みは Google Fonts の Fraunces / Archivo / Space Mono。日本語の見出しは**OS標準の明朝**を使う（読み込み不要）。

| 用途 | フォント | 例 |
|---|---|---|
| 感情的な大見出し（英字・ディスプレイ） | **Fraunces**（weight 900 or 600） | ヒーロー h1、フッター h2、`reel-title`、メイキングの英語セクション見出し `.feature h3` |
| **作品タイトル（日本語＋英数）** | **明朝スタック** weight 600 | メイキング `h1`、index `.flag-info h3`、works `.wtitle`、credits タイトル、概要パネル `.ov-title` |
| 本文・リード・説明 | **Archivo**＋日本語ゴシック | `.lead`、`.feature p`、`.about`、`.subtitle`、概要パネル本文 |
| ラベル・タグ・琥珀ラベル・ナビ・数字 | **Space Mono** | `.eyebrow`、`.wtag`/`.htag`/`.ctag`、`.nav`、`.reel-no` |

**作品タイトルの明朝スタック（コピペ用）**

```css
font-family:"Hiragino Mincho ProN","Hiragino Mincho Pro","YuMincho","Yu Mincho",serif;
font-weight:600;
letter-spacing:.01em;   /* 明朝は字間を詰めない。0〜わずかに正、マイナスにしない */
line-height:1.1〜1.18;
```

> なぜ明朝か：以前 Fraunces を日本語タイトルに当てたら英数が太く暴れた。日本語はOS明朝、英数は明朝のラテン（端正なserif）で揃える、という結論。

## 5. ★改行（line-break）ルール ★最重要・再発防止★

> このサイトで**いちばん手戻りが多かった**のがここ。新規ページを作るときは、下の3原則を必ず先に頭に入れること。チェックリストも末尾にある。

### 原則A：本文は「メディア幅」で折り返す（横幅キャップを付けない）

本文（リード・説明文）は、**画像／動画と同じ幅**で折り返させる。そうすると右端が揃って気持ちいい。

- やること：本文要素（`.lead` / `.feature p` / `.about` / `.ep-note` / `.note-txt p` 等）に **`max-width` を付けない**。
- 幅を決めてよいのは**ページの器 `.wrap` だけ**（`max-width:1100px` 等）。本文も画像もこの1本の列に収め、右端を揃える。
- ❌ ダメな例：`.feature p{max-width:64ch}` のように本文だけ狭める → 画像と右端がズレて不格好。過去のNGはこれ。
- ✅ 良い例：

```css
.wrap{max-width:1100px;margin:0 auto;padding:...}  /* ← 幅を決めるのはここだけ */
.lead{font-size:clamp(15px,1.7vw,19px)}            /* max-width 無し */
.feature p{font-size:15.5px}                        /* max-width 無し */
```

- 仕上げに `p,.lead{text-wrap:pretty}` を入れておくと、末尾の泣き別れ（1語だけ次行）が減る。

### 原則B：タイトルの改行は「意味の切れ目」で。`.nb` スパンで制御する

日本語タイトルを**ブラウザ任せで折り返すと、変な位置で切れる**（CJKはどこでも改行できてしまう）。
そこで「ここで切れてOK」という**意味のかたまり**を `<span class="nb">…</span>`（`white-space:nowrap`）で包み、**かたまり同士の間に半角スペース**を1つ入れる。スペースの位置＝唯一の改行候補になる。

```css
.nb{white-space:nowrap}
```

```html
<!-- 「PV」と「「大人序章中」」の間でだけ改行できる。1行に収まれば1行、無理なら2行 -->
<h1>
  <span class="nb">小竹竜ノ介22歳記念PV</span>
  <span class="nb">「大人序章中」</span>
</h1>
```

- 画面が広ければ1行、狭ければ「…PV」で改行して2行。**みっともない位置で割れない**。
- index の `.flag-info h3`（代表作タイトル）にも同じ `.nb` を使う。
- ❌ 生の `<br>` をタイトルに入れて固定改行にすると、スマホで早すぎる位置で割れる。タイトルは `.nb` 方式で。

### 原則C：セクション見出し（`.feature h3` など）の `<br>` は「意図した2行」だけOK

英語まじりのセクション見出しで、明確に2行に割りたいデザインのときは `<br>` を1個入れてよい。ただし**スマホで不自然にならないか必ず確認**。

```html
<h3>ロケハン × Google Map × Blenderで、<br>画を設計してから撮る</h3>
```

### 明朝タイトルの字間・行間（再掲）

- `letter-spacing` は **0〜わずかに正（.01em）**。**マイナスにしない**（明朝が窮屈に見える）。
- `line-height` は 1.1〜1.18 程度。

### 🧷 改行・再発防止チェックリスト（新規ページごとに確認）

- [ ] 本文要素に `max-width`（◯◯ch、◯◯px）を**付けていない**。幅は `.wrap` だけで決めている。
- [ ] 本文の右端が、上下の画像／動画の右端と**揃って**見える。
- [ ] メインタイトルは `.nb` スパン＋スペースで、**意味の切れ目でのみ**改行される。
- [ ] タイトルに**生の `<br>` を使っていない**（`.feature h3` の意図的2行を除く）。
- [ ] 明朝タイトルの `letter-spacing` がマイナスになっていない。
- [ ] `p,.lead{text-wrap:pretty}` が効いている。
- [ ] スマホ幅（〜380px）でタイトル・本文・見出しの折り返しを実機/プレビューで確認した。

## 6. クリック再生（YouTube ファサード）

枠は軽いポスター画像だけ置き、**クリック/Enter で初めて iframe を差し込む**（高速・埋め込み多用でも重くならない）。サイト全体で共通。

```html
<div class="frame" data-yt="動画ID" tabindex="0" role="button" aria-label="再生">
  <img class="thumb" src="ポスター.jpg" alt="...">
  <div class="play"></div>
</div>
```

```js
// data-yt を持つ枠に再生機能を付ける
document.querySelectorAll(".frame[data-yt]").forEach(f=>{
  const play=()=>{
    if(f.classList.contains("playing"))return;
    const ifr=document.createElement("iframe");
    ifr.src="https://www.youtube.com/embed/"+f.dataset.yt+"?autoplay=1&rel=0&modestbranding=1&playsinline=1";
    ifr.allow="autoplay; encrypted-media; picture-in-picture; fullscreen";
    ifr.setAttribute("allowfullscreen","");
    f.appendChild(ifr); f.classList.add("playing");
  };
  f.addEventListener("click",play);
  f.addEventListener("keydown",e=>{if(e.key==="Enter"||e.key===" "){e.preventDefault();play();}});
});
```

- ポスター画像は YouTube から：`https://img.youtube.com/vi/<ID>/maxresdefault.jpg`、無い場合に備えて `onerror` で `hqdefault.jpg` に差し替え。

```html
<img src="https://img.youtube.com/vi/<ID>/maxresdefault.jpg"
     onerror="this.onerror=null;this.src='https://img.youtube.com/vi/<ID>/hqdefault.jpg'" alt="...">
```

> 注意：作業環境のネットワークは youtube/google をブロックするので、ポスターは**ブラウザ側でのみ**読み込まれる（プレビューでは出ないことがある＝本番で確認）。

## 7. 概要ホバーパネル（index / works 共通）

カードやタイトルにカーソルを合わせると、作品概要（メイキング冒頭の内容）がフロート表示される。中身は **琥珀ラベル → タイトル → サブ → タグ → リード文 → 「メイキングを見る →」**。

- パネルのHTML・CSS・JSは index と works に**同じものが入っている**（`#ovpanel` / `.ovpanel` / `.ov-*`、`showOv`/`hideOv`/`placeOv`）。
- 挙動：トリガーの真上にフロート（上に余白が無ければ下）、画面端で自動調整、スクロール/リサイズで自動クローズ、フォーカスでも表示、`pointer-events:none`。

### index 側：カードに `data-ov-*` を付ける

```html
<a class="card" href="making-grit.html"
   data-ov-eyebrow="DOCUMENTARY — 3部作 / 2024"
   data-ov-title="GRIT3部作"
   data-ov-sub="フィジーク学生大会ドキュメンタリー"
   data-ov-tags="実写映像,ドキュメンタリー"
   data-ov-text="フィジークの学生大会に挑む友人・関根の…">
   ...
</a>
```
（`data-ov-tags` はカンマ区切り。`sub` が空なら省略可。）

### works 側：`WORKS` 配列の作品に `ov` を付ける

```js
ov:{ eyebrow:"DOCUMENTARY — 3部作 / 2024", sub:"フィジーク学生大会ドキュメンタリー",
     text:"フィジークの学生大会に挑む友人・関根の…" }
```
（`title` と `tags` は配列の既存値を流用。`ov` が無い作品＝メイキング未作成、にはパネルが出ない。）

## 12. アセット処理（作業環境メモ）

- **画像最適化**：PIL で LANCZOS リサイズ、JPEG quality 82〜88。フラッグシップは 1600×900 目安。
- **ループ動画**（メイキングの比較クリップ等）：音声なし・30fps。mp4＋webm の2本を用意。

```bash
ffmpeg -i IN.mov -an -vf "scale=1280:-2,fps=30" -c:v libx264 -crf 23 -pix_fmt yuv420p out.mp4
ffmpeg -i IN.mov -an -vf "scale=1280:-2,fps=30" -c:v libvpx-vp9 -crf 33 -b:v 0 out.webm
```

- **HEIC（iPhone写真）**：`pillow-heif` を入れて変換（`pip install pillow-heif --break-system-packages`）。
- **YouTubeサムネURL**：`https://img.youtube.com/vi/<ID>/maxresdefault.jpg`（無ければ `hqdefault.jpg`）。
- 作業環境のネットは youtube/google をブロック。github / pypi / npm は通る。
